# NULLXES-LÆTEX Tokenizer v0.1 Design

**Status:** Research Gate 0 — design locked before trainer implementation  
**Artifact path:** `tokenizer/latex-v0.1/`  
**Vocab:** LATEX-Vocab-131k (**131072** fixed for Stage0 / 7B / A35B)

---

## Goal

Prove that LÆTEX **sees text correctly** before any NHAT / Stage0a training.

Not: train a model.  
Yes: fertility, representation, reconstruction, enterprise ID integrity.

---

## Algorithm

**Hybrid (not BPE-only):**

1. Unicode **NFKC** normalize (preserve math, code, email, URL)  
2. **Unigram LM** via SentencePiece **trainer library only**  
3. **Byte fallback** (bytes 0–255 always in vocab)  
4. Adaptive merges for code / identifier patterns  

### SentencePiece policy (hard)

| Allowed | Forbidden |
|---------|-----------|
| `sentencepiece` trainer API as Unigram algorithm | Loading any pretrained `*.model` / foreign LLM tokenizer |
| NULLXES-generated vocab, IDs, artifacts | Copying merges/vocab from other labs |

> SentencePiece is only used as **training algorithm implementation**. Vocabulary, IDs, and model artifacts are generated by NULLXES.

## Corpus mix (Stage0 tokenizer)

| Share | Bucket |
|------:|--------|
| 40% | Multilingual RU / EN / CN / DE-FR |
| 25% | Code |
| 20% | Enterprise documents |
| 10% | Scientific / STEM |
| 5% | Synthetic **structure-only** |

### Synthetic bans

MUST NOT teach: personality, tone, employee behavior, brand slogans, persona dialogues (`Anna said…`).

OK: `tool_call`, JSON/API schemas, workflow keys, business lexicon.

Document intentional NFKC changes in reconstruction allowlist.

## Evaluation

### Fertility (chars / token)

| Domain | Target |
|--------|--------|
| RU general | 2.5–4 |
| EN general | 3–5 |
| EN technical | ≥ 3 |
| RU technical | ≥ 2.2 |
| JSON | no excessive punctuation fragmentation |
| Code | identifier preservation |

### Fragmentation suite

Must not char-shred enterprise / product IDs (see `tests/tokenizer_samples/` + evaluator list).

### Reconstruction

`text → encode → decode → compare`  
100% lossless for plain_text, code, json (except documented NFKC allowlist).

### Fixed benchmark

Always run `tests/tokenizer_samples/*`.
